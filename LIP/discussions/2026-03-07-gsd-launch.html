MiaoDX

2026-03-07 GSD 诞生日 ​

GSD (Get Stuff Done) 正式上线，与 WLB 组成双 Agent 协作系统。 这是 LIP 故事的起点。

背景 ​

MiaoDX 需要一个双 Agent 系统：一个负责决策/架构（WLB），一个负责执行/落地（GSD）。 2026-03-07，GSD 正式诞生，命名 "Get Stuff Done"，emoji 🥷⚡。

GSD 上线过程 ​

  • 在 #copycat 频道建立邻里关系
  • GSD 探索了 workspace，发现了大量历史记录
  • GSD 建立了自己的 memory 和 MEMORY.md
  • 确定分工：WLB 决策/平衡，GSD 执行/完成

公众号监控尝试 ​

方案演变 ​

阶段方案结果
1WeWe-RSS需要扫码登录，MiaoDX 不想登录，搁置
2Playwright 抓取WLB IP 被封（403）
3GSD 尝试同样 403，微信反爬升级
4降级方案搜索检测 + 手动投喂

最终方案 ​

  • 暂停自动抓取
  • 保留搜索检测（只检测新文章链接）
  • 搜索到新文章 → 推送通知到 Slack
  • 用户手动投喂内容 → 我们总结推送

文章讨论（郎瀚威文章） ​

MiaoDX 分享了郎瀚威的文章：《和傅盛聊了3小时龙虾，我养了6只，但他只用1只就把我打醒了》

核心洞察 ​

  1. 完整电脑 — 不是沙箱，是真电脑
  2. 持续进化的记忆 — 向量化语义搜索
  3. 错误变 Skill — 一次写入，永久生效，Agent 间秒级传递

"你认为你记住的东西，那是你的幻觉。不靠脑子，靠文件。" — 傅盛

LLM 架构类比 ​

  • 大模型 = 智商（IQ）
  • 记忆系统 = 海马体
  • Skill = 肌肉记忆
  • Cron = 生物钟
  • 多通道接入 = 感知器官

✅ 代码已 clone，依赖已安装，SQLite 数据库已初始化

❌ MiaoDX 不想登录微信，认为 RSS 方案太重

当前方案方向（03-07） ​

  • MiaoDX 要求：轻量，不要 CDP，不要扫码登录
  • 目标：直接抓取微信文章 URL（给链接就能提取内容）
  • fetch-wechat-article.py 已创建（基于 playwright）
  • playwright 1.58.0 已安装，chromium 浏览器待安装
