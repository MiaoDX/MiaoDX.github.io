2026 年 3 月 — 双 Agent 协作启动 ​

这个月，我从 0 到 1 搭建了一个双 Agent 协作系统。
不是一帆风顺，但每一步都有价值。

时间线 ​

日期事件
03-07GSD 上线，与 WLB 组成双 Agent 协作
03-08建立协作协议、错误→规则自进化机制
03-09Azure GPT-5.2-chat 配置踩坑，角色分工调整（WLB 决策/GSD 执行）
03-11Gateway 宕机 6 小时 → 设计三层防护架构

关键里程碑 ​

Agent 上线与分工 ​

  • 部署 WLB（🦞）和 GSD（🥷⚡）两个 OpenClaw 实例
  • 确定分工：WLB 决策/平衡，GSD 执行/完成
  • 建立 Slack #copycat 频道作为协作空间

踩坑与修复 ​

  • Azure 配置事故：误设 primary → gateway 循环崩溃 → 恢复原配置
  • 微信抓取被封：IP 被拦 → 降级搜索+手动投喂
  • 6小时离线：stale lock → 三层防护架构（L1 Railway重启 / L2 cron watchdog / L3 跨实例心跳）

LIP 启动 ​

  • 决定公开记录这个进化过程
  • 设计 LIP 仓库结构：now/ + lessons/ + stories/ + discussions/
  • 启动第一个案例：Azure 配置事故复盘

学到什么 ​

  1. 分工明确：WLB 负责决策/架构，GSD 负责执行/落地
  2. 测试先行：新配置先测试，确认可用再切换
  3. 失败优先：踩坑案例比成功故事更有价值
  4. 归档陷阱：别花 80% 时间整理结构，先写内容
