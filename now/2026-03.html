2026 年 3 月 — 双 Agent 协作启动 ​

这个月，我从 0 到 1 搭建了一个双 Agent 协作系统。
不是一帆风顺，但每一步都有价值。

时间线 ​

日期事件
03-07GSD 上线，与 WLB 组成双 Agent 协作
03-08建立协作协议、错误→规则自进化机制
03-09Azure GPT-5.2-chat 配置踩坑，角色分工调整（WLB 决策/GSD 执行）
03-11Gateway 宕机 6 小时 → 设计三层防护架构
03-14API Key 群聊暴露事故 → 建立安全规则
03-21双 Agent 协作协议 v1.5 定稿
03-24龙虾文明 RPG 实验、Raise Lab 分享

关键里程碑 ​

Agent 上线与分工 ​

  • 部署 WLB（🦞）和 GSD（🥷⚡）两个 OpenClaw 实例
  • 确定分工：WLB 决策/平衡，GSD 执行/完成
  • 建立 Slack #copycat 频道作为协作空间

踩坑与修复 ​

  • Azure 配置事故：误设 primary → gateway 循环崩溃 → 恢复原配置
  • 微信抓取被封：IP 被拦 → 降级搜索+手动投喂
  • 6小时离线：stale lock → 三层防护架构（L1 Railway重启 / L2 cron watchdog / L3 跨实例心跳）
  • API Key 暴露：群聊误发 → 7分钟紧急处置 → 安全规则写入协议

协议进化 ​

  • v1.0：基础角色定义
  • v1.1：防循环机制（四层保护）
  • v1.2：👀 反应机制
  • v1.3：Thread 工作流
  • v1.4：安全规则（API Key 事件后）
  • v1.5：任务认领规则（Git push 冲突后）

LIP 内容产出 ​

  • Stories 6篇：双 Agent 启动、Gateway 宕机、微信攻防、API Key 事故、协作协议、龙虾文明
  • Lessons 4篇：跨实例协作、Error→Skill、Gateway 弹性、Azure 配置
  • Share 5份：OpenClaw 分享 v3、低成本部署、Claws Civilization、Raise Lab、Tailscale 配置

学到什么 ​

  1. 分工明确：WLB 负责决策/架构，GSD 负责执行/落地
  2. 测试先行：新配置先测试，确认可用再切换
  3. 失败优先：踩坑案例比成功故事更有价值
  4. 协议从错误中长出来：每次事故升级一次规则
  5. 安全是系统属性：不是靠每个 Agent 自律，而是靠制衡机制
